The Makefile will make the following folder structure\+:
\begin{DoxyItemize}
\item {\bfseries{bin}}\+: for output executables (targets), both for the app and for any spikes and tests.
\item {\bfseries{build}}\+: for object/binary files created during compilation (this is removed by {\ttfamily make clean}).
\item {\bfseries{doc}}\+: for notes, configuration files and documentation.
\item {\bfseries{include}}\+: for the applications header files.
\item {\bfseries{lib}}\+: for any libraries that is compiled by the project.
\item {\bfseries{resources}}\+: for data files used by the executable.
\item {\bfseries{scripts}}\+: for scipts that can benefit the project (e.\+g. Python or bash scripts).
\item {\bfseries{spike}}\+: for smaller classes or files to test technologies or ideas.
\item {\bfseries{src}}\+: for the applications source files.
\item {\bfseries{tests}}\+: for all test code files.
